# TypeScript Roadmap – 6 Weeks

## Weeks 1-2: Foundations (🟢)
- [ ] Day 1: Install TS; understand tsconfig basics
- [ ] Day 2: Primitives, arrays, tuples, enums
- [ ] Day 3: Functions, params, returns, narrowing
- [ ] Day 4: Unions/intersections; literal types
- [ ] Day 5: Interfaces vs type aliases; structural typing
- [ ] Day 6: Basic generics; constraints
- [ ] Day 7: strict mode flags (noImplicitAny, strictNullChecks)
- [ ] Day 8: Modules/imports; path aliases
- [ ] Day 9: Basic tooling (ts-node, ts-jest)
- [ ] Day 10: Mini-project: typed utilities
- [ ] Day 11-12: Lint/format setup (ESLint/Prettier)
- [ ] Day 13-14: Review + refactor

**Milestone**: Comfortable writing typed functions and modules with strict mode.

## Weeks 3-4: Intermediate (🟡)
- [ ] Day 15: Advanced generics; utility types
- [ ] Day 16: Discriminated unions; exhaustive switches
- [ ] Day 17: Mapped types; keyof/typeof inference
- [ ] Day 18: Declaration merging; ambient types
- [ ] Day 19: Runtime validation (zod/yup) + type inference
- [ ] Day 20: Working with APIs; typed fetch wrappers
- [ ] Day 21: Error handling patterns; Result/Either types
- [ ] Day 22: Testing with types (ts-jest/ts-node)
- [ ] Day 23: Config targets (lib/module/moduleResolution)
- [ ] Day 24-25: Mini-project: typed API client + tests
- [ ] Day 26-28: Performance: incremental builds; isolatedModules

**Milestone**: Production-ready typing patterns; robust DTOs and clients.

## Weeks 5-6: Advanced (🔴)
- [ ] Day 29: Project references; monorepo setup
- [ ] Day 30: Library authoring; d.ts packaging
- [ ] Day 31: Advanced inference tricks; conditional types
- [ ] Day 32: Strict DX rules; ban implicit any/misused promises
- [ ] Day 33: Domain modeling; branded types; opaque IDs
- [ ] Day 34: Build optimization; emit targets; bundler configs
- [ ] Day 35: Governance: lint rules, CI checks, code owners
- [ ] Day 36: Security: avoid unsafe any, sanitize external data
- [ ] Day 37-38: Capstone: typed SDK with runtime validation
- [ ] Day 39-42: Documentation, handover, patterns guide

**Milestone**: Enterprise-grade TS practices, scalable builds, governed quality.
