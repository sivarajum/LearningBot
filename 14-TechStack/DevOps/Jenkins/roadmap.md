# Jenkins Roadmap – 6 Weeks

## Weeks 1-2: Foundations (🟢)
- [ ] Day 1: Install Jenkins (Docker/LTS); admin unlock
- [ ] Day 2: Freestyle jobs; parameters; artifacts
- [ ] Day 3: Declarative pipeline basics; stages/steps
- [ ] Day 4: Jenkinsfile in repo; multibranch pipeline
- [ ] Day 5: Triggers (webhook/poll/cron)
- [ ] Day 6: Credentials store; mask secrets
- [ ] Day 7: Agents overview; labels; executors
- [ ] Day 8: Notifications (email/slack)
- [ ] Day 9: Workspace management; cleanup
- [ ] Day 10: Basic security (users, matrix auth)
- [ ] Day 11-12: Mini-project: build+test pipeline
- [ ] Day 13-14: Review + harden basics

**Milestone**: Jenkinsfile-driven pipelines; basic security.

## Weeks 3-4: Intermediate (🟡)
- [ ] Day 15: Shared libraries; reuse steps
- [ ] Day 16: Parallel stages; post conditions
- [ ] Day 17: Artifact retention; fingerprinting
- [ ] Day 18: Caching strategies; pipeline durability
- [ ] Day 19: Docker agents; containerized builds
- [ ] Day 20: Kubernetes agents (if available)
- [ ] Day 21: Quality gates: tests, lint, coverage, SCA
- [ ] Day 22: Promotions/dev→stage→prod with approvals
- [ ] Day 23: Credentials best practices; vault integration
- [ ] Day 24: Backup/restore JENKINS_HOME; config-as-code
- [ ] Day 25-26: Mini-project: multi-stage CI/CD with approvals
- [ ] Day 27-28: Harden controller (HTTPS, RBAC)

**Milestone**: Reusable libraries, gated pipelines, secure basics.

## Weeks 5-6: Advanced (🔴)
- [ ] Day 29: HA/DR options; controller redundancy strategy
- [ ] Day 30: Observability: metrics, logs, audit
- [ ] Day 31: Plugin governance; minimal set; update policy
- [ ] Day 32: Policy enforcement (security scans, license checks)
- [ ] Day 33: Supply chain: signed artifacts/images
- [ ] Day 34: Self-service patterns; folder isolation
- [ ] Day 35: Performance tuning (agents/executors, queue mgmt)
- [ ] Day 36: Cost controls (agent lifecycle, caching)
- [ ] Day 37: Incident runbooks; break-glass procedures
- [ ] Day 38: Blue/green pipelines; rollback patterns
- [ ] Day 39-40: Capstone: enterprise pipeline with gates + DR tested
- [ ] Day 41-42: Documentation + handover

**Milestone**: Enterprise-ready Jenkins with governance and DR.
