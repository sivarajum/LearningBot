# Learning Paths Index

## Paths
- **Data Engineer on GCP** (3w): GCS → BQ → Dataflow → Pub/Sub → Airflow/dbt → Monitoring.
- **Backend Engineer on GCP** (3w): Cloud Build → Cloud Run → Cloud SQL/Spanner → VPC/IAM → LB → Observability.
- **MLOps/LLMOps Engineer** (4w): Vertex/AutoML/BQML → Pipelines/Registry → Monitoring → RAG/LLMs.
- **Full DevOps Stack** (4w): Docker → Kubernetes → Terraform → CI/CD (GHA/Jenkins) → Monitoring/Security.
- **Data Engineering Stack** (4w): Spark → Kafka → Airflow → dbt → Data Quality/Observability.

## Prerequisites
- Common: Git, basic Linux, editor.
- Data tracks: Python + SQL.
- Backend: Python/Node + REST + basics of HTTP.
- DevOps: Docker basics; willingness for infra/policies.
- MLOps: Python + ML basics; some SQL.

## Selection Guide
- Want cloud data pipelines? **Data Engineer on GCP**.
- Shipping APIs on GCP? **Backend Engineer on GCP**.
- Shipping ML/LLM to prod? **MLOps/LLMOps**.
- Platform/infra focus? **Full DevOps Stack**.
- Broad data engineering (non cloud-specific)? **Data Engineering Stack**.

- 3-week tracks: ~2–3 hrs/day.
- 4-week tracks: ~2–3 hrs/day, plus extra for capstone.

## Milestones per Track (summary)
- Week 1: Foundations + first mini-project.
- Week 2: Integration + orchestration/security.
- Week 3: Observability, cost, governance, capstone build (for 3w).
- Week 4 (if present): Advanced (RAG/LLM or DevOps hardening) + capstone.
