## 🟢 BASIC LEVEL Questions

### Q1: What is Databricks?

**Answer:**
"Databricks is a unified analytics platform built on Apache Spark. It provides a collaborative environment for data engineering, data science, and ML.

Key features:
- Optimized Spark (faster than open-source)
- Delta Lake (ACID transactions)
- Unity Catalog (data governance)
- MLflow integration
- Collaborative notebooks

I use Databricks for data engineering and ML, leveraging Delta Lake for reliable data processing and MLflow for experiment tracking."

**Key Points:**
- Unified platform
- Optimized Spark
- Delta Lake
- MLflow integration

---

### Q2: What is Delta Lake?

**Answer:**
"Delta Lake is an open-source storage layer that brings ACID transactions to data lakes.

**Features:**
- ACID transactions
- Time travel (version history)
- Schema evolution
- Better performance

**Benefits:**
- Data reliability
- Concurrent writes
- Rollback capability
- Faster queries

I use Delta Lake for reliable data processing, enabling ACID transactions and time travel for data quality."

**Key Points:**
- ACID transactions
- Time travel
- Schema evolution
- Better performance

## 🟡 INTERMEDIATE LEVEL Questions

### Q3: How does Databricks differ from open-source Spark?

**Answer:**
"**Databricks Optimizations:**
- **Photon Engine**: Vectorized execution (10x faster)
- **Delta Lake**: ACID transactions
- **Unity Catalog**: Data governance
- **MLflow**: Built-in ML lifecycle
- **Collaboration**: Team features

**Performance:**
- 10-100x faster queries
- Better resource management
- Optimized I/O

**Enterprise Features:**
- Security and governance
- Collaboration tools
- Managed infrastructure

I use Databricks for production workloads because of better performance, reliability, and enterprise features."

**Key Points:**
- Optimized Spark
- Enterprise features
- Better performance
- Managed infrastructure
