# FastAPI Roadmap – 6 Weeks

## Weeks 1-2: Foundations (🟢)
- [ ] Day 1: Install FastAPI + uvicorn; first endpoint
- [ ] Day 2: Path/query/body params; Pydantic models
- [ ] Day 3: Response models; status codes; error handling
- [ ] Day 4: Dependencies; settings/config pattern
- [ ] Day 5: Routers and modularization
- [ ] Day 6: Auto docs; OpenAPI basics
- [ ] Day 7: Mini-project: CRUD for one resource
- [ ] Day 8: Testing basics with pytest + httpx
- [ ] Day 9: Logging baseline
- [ ] Day 10: Review + refactor
- [ ] Day 11-12: Type checking (mypy); linting (ruff/black)
- [ ] Day 13-14: Env handling; .env; settings class

**Milestone**: Clean CRUD API with tests and linting.

## Weeks 3-4: Intermediate (🟡)
- [ ] Day 15: Auth basics: OAuth2/JWT; password hashing
- [ ] Day 16: Permissions/scopes; dependency-based authz
- [ ] Day 17: DB layer: async driver + repositories
- [ ] Day 18: Services/use-case layer; DI
- [ ] Day 19: Error patterns; global handlers; validation errors
- [ ] Day 20: Pagination; filtering; ordering
- [ ] Day 21: Background tasks; async vs Celery/RQ
- [ ] Day 22-23: Mini-project: auth + CRUD + async DB
- [ ] Day 24-25: Observability: request IDs, structured logs, metrics
- [ ] Day 26-28: CORS; security headers; rate limiting

**Milestone**: Authenticated API with DB, observability, and safeguards.

## Weeks 5-6: Advanced (🔴)
- [ ] Day 29: Tracing with OpenTelemetry; Prometheus metrics
- [ ] Day 30: Performance profiling; async pitfalls
- [ ] Day 31: API versioning; breaking-change strategy
- [ ] Day 32: Contracts: response models, error schemas
- [ ] Day 33: CI/CD: lint/format/type/test on PR; build image
- [ ] Day 34: Security hardening: headers, SSRF guard, input sanitization
- [ ] Day 35: Load testing; capacity planning
- [ ] Day 36: Blue/green or canary deploy patterns
- [ ] Day 37-38: Capstone: production-ready service with docs/runbooks
- [ ] Day 39-42: Documentation and handover

**Milestone**: Production-grade FastAPI service with CI/CD and observability.
