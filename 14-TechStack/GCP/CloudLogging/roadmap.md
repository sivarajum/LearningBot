# Cloud Logging Roadmap – 6 Weeks

## Weeks 1-2: Foundations (🟢)
- [ ] Day 1: Explore Logs Explorer; resources/severity
- [ ] Day 2: Structured logging best practices
- [ ] Day 3: Create custom log bucket; set retention
- [ ] Day 4: Create sink to Storage/BigQuery
- [ ] Day 5: Basic filters; severity/error queries
- [ ] Day 6: Views for scoped access
- [ ] Day 7: Mini-project: structured app logs + sink
- [ ] Day 8: IAM for logging buckets/sinks
- [ ] Day 9: Log-based metrics intro
- [ ] Day 10: Review + cleanup
- [ ] Day 11-12: Audit logs awareness
- [ ] Day 13-14: Cost/noise basics (filters, sampling)

**Milestone**: Structured logs, retention, sink, and basic metrics in place.

## Weeks 3-4: Intermediate (🟡)
- [ ] Day 15: Org/folder sinks; aggregated exports
- [ ] Day 16: VPC-SC/CMEK considerations
- [ ] Day 17: Log-based metrics for errors/latency
- [ ] Day 18: Alerts via Monitoring on log metrics
- [ ] Day 19: Views for multi-tenant access control
- [ ] Day 20: Pub/Sub export to SIEM/ELK
- [ ] Day 21: Noise reduction; drop/debug filtering
- [ ] Day 22-23: Mini-project: central logging with BQ/SIEM export
- [ ] Day 24-25: Performance/cost review of logs and metrics
- [ ] Day 26-28: Governance: standard log schema; labels

**Milestone**: Centralized, governed logging with metrics and alerts.

## Weeks 5-6: Advanced (🔴)
- [ ] Day 29: Access transparency/audit log retention strategy
- [ ] Day 30: DR for logging data; export/backup cadence
- [ ] Day 31: Advanced filters; sampling strategies
- [ ] Day 32: Compliance reviews; access reviews
- [ ] Day 33: SLOs for error rates; dashboards
- [ ] Day 34: Integration with tracing/metrics; correlation IDs
- [ ] Day 35: Runbooks for noisy logs; budget alarms
- [ ] Day 36: CI checks for log schema adherence
- [ ] Day 37-38: Capstone: enterprise logging with SIEM integration
- [ ] Day 39-42: Documentation, SOPs, audits

**Milestone**: Enterprise logging with governance, SIEM, and SLOs.
