# BigQuery Roadmap – 6 Weeks

## Weeks 1-2: Foundations (🟢)
- [ ] Day 1: Project setup; IAM basics; bq CLI
- [ ] Day 2: Datasets/tables/views; standard SQL
- [ ] Day 3: Partitioning (date/timestamp/ingestion)
- [ ] Day 4: Clustering; pruning effects
- [ ] Day 5: Loading data (CSV/Parquet/ORC); schema handling
- [ ] Day 6: External tables (GCS); storage formats
- [ ] Day 7: Query plans; bytes processed; avoid SELECT *
- [ ] Day 8: Scheduled queries basics
- [ ] Day 9: Mini-project: load + query + scheduled report
- [ ] Day 10: Review + refactor
- [ ] Day 11-12: Cost basics; on-demand pricing; slot intro
- [ ] Day 13-14: Time travel and snapshots

**Milestone**: Confident loading/querying with partitions/clusters.

## Weeks 3-4: Intermediate (🟡)
- [ ] Day 15: Materialized views; incremental patterns
- [ ] Day 16: Performance tuning: filters, clustering keys
- [ ] Day 17: Slot reservations; BI Engine basics
- [ ] Day 18: Authorized views; row/column security
- [ ] Day 19: Schema evolution; breaking change strategy
- [ ] Day 20: Data quality checks; staging vs prod datasets
- [ ] Day 21: Export/import; DR basics
- [ ] Day 22-23: Mini-project: staged pipeline with MVs
- [ ] Day 24-25: Monitoring: slot usage, query latency, errors
- [ ] Day 26-28: Optimize costs: storage tiers, partitions, slots

**Milestone**: Production-ready modeling with security and cost controls.

## Weeks 5-6: Advanced (🔴)
- [ ] Day 29: Advanced SQL (window functions, arrays/structs)
- [ ] Day 30: Streaming vs load; latency vs cost tradeoffs
- [ ] Day 31: Governance: audit logs, policies, lineage docs
- [ ] Day 32: DR/backup: exports, snapshots, retention
- [ ] Day 33: Integration: Dataflow/DBT/Composer orchestration
- [ ] Day 34: Reliability: quotas/limits handling; retries/backoff
- [ ] Day 35: BI performance: clustering review, BI Engine, caches
- [ ] Day 36: Compliance: CMEK, row/col security, access reviews
- [ ] Day 37-38: Capstone: end-to-end pipeline with SLOs and monitoring
- [ ] Day 39-42: Documentation, runbooks, cost/perf reports

**Milestone**: Governed, observable, cost-optimized BigQuery pipelines.
