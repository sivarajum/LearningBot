GCP Core Data Services — Part 1
1
BigQuery
Serverless Data Warehouse
What is it?
Fully managed, serverless data warehouse for analytics using SQL. Petabyte-scale with built-in ML & BI integration.
Key Features
  • Serverless, no infra mgmt
  • Columnar (Dremel engine)
  • ANSI SQL support
  • Storage/compute separation
  • ML & GIS integrated
  • Petabyte scale
Best Use Cases
  • Analytics & reporting
  • BI dashboards
  • Ad-hoc queries
  • Data warehousing
  • ML analytics
Architecture / How it works
Databases

Files (GCS)
→

→
BI Tools
→
ML/AI
✅ Best Practices
  • Partition by date/timestamp
  • Cluster frequently filtered cols
  • Avoid SELECT *
  • Use materialized views
  • Schedule queries for automation
❌ When Not To Use
  • Not for OLTP workloads
  • Not for <1s latency needs
  • Avoid for transactional data
  • Not for small datasets
Exam Tip: For analytical SQL workloads at scale, choose BigQuery.

Dataproc
Managed Hadoop & Spark Service
What is it?
Managed service for running Hadoop, Spark, Hive and other open-source big data tools on Google Cloud infrastructure.
Key Features
  • Managed clusters
  • Supports Spark, Hive, HBase, Pig, Presto
  • Custom init actions
  • Autoscaling clusters
  • Preemptible VMs
  • Integrates with GCS, BigQuery
Best Use Cases
  • Lift & shift Hadoop
  • Existing Spark processing
  • IoT transformation
  • ML model training
  • Data lake processing
Architecture / How it works

→

GCS

✅ Best Practices
  • Use autoscaler
  • Enable autoscaling
  • Use preemptible VMs
  • Store data in GCS (not HDFS)
  • Right-size machine types
❌ When Not To Use
  • Need serverless solution
  • New pipelines → use Dataflow
  • Minimal ops required

Cloud Storage (GCS)
Object Storage
What is it?
Highly durable, scalable object storage for data lakes, backups, archives and more. 99.999999999% durability.
Key Features
  • 99.999999999% durability
  • Global & regional policies
  • Lifecycle management
  • Versioning
  • Signed URLs
  • Object-level IAM
Storage Classes
  • Standard – Frequent access
  • Nearline – Monthly access
  • Coldline – Quarterly
  • Archive – Long-term
Architecture / How it works
Apps
→
Upload/Download
→
GCS Bucket
→
Dataflow / BQ
✅ Best Practices
  • Use appropriate storage class
  • Configure lifecycle policies
  • Enable versioning if needed
  • Use folder structure & permissions
❌ When Not To Use
  • Not for RDBMS workloads
  • Not for low-latency queries
  • Avoid frequent data egress charges
Exam Tip: Foundation of data lake on GCP. Use lifecycle to save cost.
