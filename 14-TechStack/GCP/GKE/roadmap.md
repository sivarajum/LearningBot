# GKE Roadmap – 6 Weeks

## Weeks 1-2: Foundations (🟢)
- [ ] Day 1: Autopilot vs Standard; create cluster
- [ ] Day 2: kubectl basics; Deployments/Services
- [ ] Day 3: Ingress + LB; health checks
- [ ] Day 4: Pods/ReplicaSets; readiness/liveness
- [ ] Day 5: Namespaces; RBAC basics
- [ ] Day 6: Workload Identity intro
- [ ] Day 7: Mini-project: public service with LB
- [ ] Day 8: Logging/monitoring basics (Cloud Ops)
- [ ] Day 9: Node pools basics; autoscaling intro
- [ ] Day 10: Review + tidy configs
- [ ] Day 11-12: Network basics; private vs public clusters
- [ ] Day 13-14: Security basics: Pod Security levels

**Milestone**: Running workloads with LB and basic security/observability.

## Weeks 3-4: Intermediate (🟡)
- [ ] Day 15: Regional clusters; surge upgrades; maintenance windows
- [ ] Day 16: Node pools per workload; spot/preemptible pools
- [ ] Day 17: NetworkPolicies; master authorized networks
- [ ] Day 18: Workload Identity deep dive; avoid node creds
- [ ] Day 19: Ingress patterns: GCLB/ILB/NEGs
- [ ] Day 20: HPA/VPA; PDBs; quotas/limits
- [ ] Day 21: Binary Authorization/image scanning
- [ ] Day 22-23: Mini-project: private cluster + Ingress + WI
- [ ] Day 24-25: Observability: traces/metrics/logs, audit
- [ ] Day 26-28: Cost controls: right-size nodes, autoscaler tuning

**Milestone**: Secure, autoscaled, observable cluster.

## Weeks 5-6: Advanced (🔴)
- [ ] Day 29: OPA/Kyverno policies; governance
- [ ] Day 30: Service mesh (Istio/Linkerd) for traffic policy/MTLS
- [ ] Day 31: Multi-cluster/multi-region strategy; hub/spoke
- [ ] Day 32: DR runbooks; backup/restore; Velero
- [ ] Day 33: Performance/load tests; tune requests/limits
- [ ] Day 34: SLOs, alerts; cost dashboards
- [ ] Day 35: Release strategy: canary/blue-green
- [ ] Day 36: Supply chain: signed images, provenance
- [ ] Day 37-38: Capstone: regional private GKE with policies and mesh
- [ ] Day 39-42: Documentation, SOPs, audits

**Milestone**: Production-grade, governed, multi-tenant GKE cluster.
