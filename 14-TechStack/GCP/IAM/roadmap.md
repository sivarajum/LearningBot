# IAM Roadmap – 6 Weeks

## Weeks 1-2: Foundations (🟢)
- [ ] Day 1: Principals, roles, policies; resource hierarchy
- [ ] Day 2: Grant/revoke roles; gcloud basics
- [ ] Day 3: Service accounts; keyless vs keys
- [ ] Day 4: Predefined vs basic vs custom roles
- [ ] Day 5: Least privilege principle; scopes
- [ ] Day 6: Audit logs overview
- [ ] Day 7: Mini-project: lock down a project with least privilege
- [ ] Day 8: Access transparency/audit awareness
- [ ] Day 9: Folder/project structure basics
- [ ] Day 10: Review + cleanup
- [ ] Day 11-12: Intro to org policies
- [ ] Day 13-14: Access review cadence design

**Milestone**: Project with least privilege and audit visibility.

## Weeks 3-4: Intermediate (🟡)
- [ ] Day 15: Service account governance; disable key creation
- [ ] Day 16: WIF for CI/CD; short-lived creds
- [ ] Day 17: Custom roles for tight scopes
- [ ] Day 18: Conditional IAM policies (context-aware)
- [ ] Day 19: Org policies: restrict domains, external IPs
- [ ] Day 20: VPC-SC intro for data exfil control
- [ ] Day 21: Alerts on IAM changes; log sinks
- [ ] Day 22-23: Mini-project: WIF CI pipeline with least privilege
- [ ] Day 24-25: Break-glass process and audit
- [ ] Day 26-28: Periodic access review automation

**Milestone**: Governed IAM with WIF, org policies, alerting.

## Weeks 5-6: Advanced (🔴)
- [ ] Day 29: Enterprise folder/org structure; env separation
- [ ] Day 30: Policy-as-code with Terraform; drift detection
- [ ] Day 31: VPC-SC policies for key data services
- [ ] Day 32: Compliance alignment (SOC2/ISO); evidence collection
- [ ] Day 33: Key management lifecycle; rotation and attestation
- [ ] Day 34: Least privilege enforcement checks in CI
- [ ] Day 35: Incident response for IAM changes
- [ ] Day 36: Continuous monitoring dashboards for IAM
- [ ] Day 37-38: Capstone: org-wide IAM baseline with reviews
- [ ] Day 39-42: Documentation, SOPs, audit prep

**Milestone**: Enterprise IAM with policy-as-code, WIF, monitoring, compliance.
