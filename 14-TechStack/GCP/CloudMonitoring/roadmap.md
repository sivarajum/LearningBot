# Cloud Monitoring Roadmap – 6 Weeks

## Weeks 1-2: Foundations (🟢)
- [ ] Day 1: Metrics explorer; dashboards basics
- [ ] Day 2: Notification channels; policies overview
- [ ] Day 3: Create simple alert (CPU/latency)
- [ ] Day 4: Uptime checks for an endpoint
- [ ] Day 5: Log-based metrics intro
- [ ] Day 6: Mini-project: service dashboard + alert
- [ ] Day 7: Review + tidy alerts
- [ ] Day 8: Understand RED/USE metrics
- [ ] Day 9: SLI/SLO concepts
- [ ] Day 10: Review cardinality risks
- [ ] Day 11-12: On-call routing; test notifications
- [ ] Day 13-14: Cost basics for metrics/alerts

**Milestone**: Basic dashboards, uptime check, alerts with clean channels.

## Weeks 3-4: Intermediate (🟡)
- [ ] Day 15: Multi-window burn-rate alerts
- [ ] Day 16: Service dashboards per team; ownership labels
- [ ] Day 17: Log-based metrics for errors/latency
- [ ] Day 18: Synthetic probes for key flows
- [ ] Day 19: GKE/GCE/GCF/GCR built-in metrics
- [ ] Day 20: Correlate logs/metrics/traces
- [ ] Day 21: Alert noise reduction; silence policies
- [ ] Day 22-23: Mini-project: SLO + burn-rate alert
- [ ] Day 24-25: Integrate with on-call (PagerDuty/etc.)
- [ ] Day 26-28: Cost controls; prune unused metrics/alerts

**Milestone**: SLO-driven alerting with ownership and noise control.

## Weeks 5-6: Advanced (🔴)
- [ ] Day 29: Org/folder dashboards; governance
- [ ] Day 30: Channel management at scale; access reviews
- [ ] Day 31: Incident runbooks linked in alerts
- [ ] Day 32: Error budgets and policies; reporting
- [ ] Day 33: Performance tests of alert latency
- [ ] Day 34: Compliance: audit alert/policy changes
- [ ] Day 35: DR for observability (backups/exports as needed)
- [ ] Day 36: Automation via Terraform/API
- [ ] Day 37-38: Capstone: org-wide SLOs + burn alerts + runbooks
- [ ] Day 39-42: Documentation, SOPs, dashboards hygiene

**Milestone**: Enterprise observability with SLO governance and low noise.
