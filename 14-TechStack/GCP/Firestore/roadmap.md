# Firestore Roadmap – 6 Weeks

## Weeks 1-2: Foundations (🟢)
- [ ] Day 1: Create DB; understand native vs datastore mode
- [ ] Day 2: CRUD basics; collections/docs
- [ ] Day 3: Queries; filters; ordering; cursors
- [ ] Day 4: Indexes (single/composite)
- [ ] Day 5: Security rules basics
- [ ] Day 6: SDK usage (server vs client)
- [ ] Day 7: Mini-project: simple app with rules
- [ ] Day 8: Costs: reads/writes/doc/listen
- [ ] Day 9: Exports to GCS
- [ ] Day 10: Review + refine rules
- [ ] Day 11-12: Sharding patterns; random IDs
- [ ] Day 13-14: TTL policies; cleanup tasks

**Milestone**: Secure basic app with indexes and cost awareness.

## Weeks 3-4: Intermediate (🟡)
- [ ] Day 15: Complex queries; composite indexes tuning
- [ ] Day 16: Hotspot mitigation; fan-out design
- [ ] Day 17: Backup/restore via exports; schedules
- [ ] Day 18: Latency and read reduction (select/limits)
- [ ] Day 19: Rules staging and versioning
- [ ] Day 20: Monitoring/alerts on reads/writes/latency
- [ ] Day 21: Triggers via Cloud Functions
- [ ] Day 22-23: Mini-project: event-driven workflow
- [ ] Day 24-25: Data modeling review; flattening
- [ ] Day 26-28: Cost controls: remove unused indexes; TTL more data

**Milestone**: Hardened rules, optimized queries, scheduled backups.

## Weeks 5-6: Advanced (🔴)
- [ ] Day 29: Multi-region setup; consistency tradeoffs
- [ ] Day 30: DR drills; restore tests
- [ ] Day 31: App Check; auth strategies
- [ ] Day 32: Governance: audit logs, access reviews
- [ ] Day 33: Observability dashboards; error budgets
- [ ] Day 34: Data export to BQ for analytics
- [ ] Day 35: CI/CD for rules + indexes
- [ ] Day 36: Performance testing at scale
- [ ] Day 37-38: Capstone: event-driven, multi-region-ready app
- [ ] Day 39-42: Documentation, SOPs, runbooks

**Milestone**: Production Firestore with DR, governance, and cost controls.
