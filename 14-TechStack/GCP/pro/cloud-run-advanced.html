Cloud Run Overview

Basic

Cloud Run is Google's fully managed serverless platform for running containerized applications. It automatically scales your applications based on traffic, from zero to millions of requests, and you only pay for the compute time you use.

✅ Cloud Run Use Cases
  • Web Applications: React, Vue, Angular apps with server-side rendering
  • REST APIs: Backend services for mobile apps and SPAs
  • Microservices: Decomposed applications with independent scaling
  • Data Processing: ETL pipelines, image processing, ML inference
  • Development Tools: CI/CD helpers, code generators, dev servers
⚠️ When NOT to Use Cloud Run
  • Long-running Tasks: Use Cloud Run Jobs for batch processing >15min
  • Stateful Applications: Consider GKE for persistent state requirements
  • High-performance Computing: Use Compute Engine for GPU workloads
  • Legacy Applications: May require containerization effort
  • Real-time Requirements: Consider cold start latency for sub-second SLAs

17 """Handle Cloud Storage events via Eventarc."""
18 try:

20
21 # Validate Eventarc event
22 if not envelope or 'message' not in envelope:
23 return jsonify({'error': 'Invalid event format'}), 400
24

28 pubsub_message['data'].decode('utf-8')
29 )
30

33
