VPC Network Overview

Basic

Virtual Private Cloud (VPC) provides networking functionality to Compute Engine virtual machine instances, Kubernetes Engine clusters, and App Engine Flex applications. It offers global, scalable, and flexible networking for your cloud-based services.

Subnets & IP Addressing

Basic

VPC networks are subdivided into regional subnets, each with its own IP address range. You can create subnets in auto mode (automatic) or custom mode (manual) depending on your needs.

🤖
Auto Mode
Automatic subnet creation with predefined IP ranges per region
⚙️
Custom Mode
Manual subnet creation with custom IP ranges and sizes
🔒
Private Subnets
Subnets without external IP addresses for enhanced security
🌍
Shared VPC
VPC networks shared across multiple projects in an organization
1# Create auto mode VPC network
2gcloud compute networks create my-auto-network \
3 --subnet-mode=auto \
4 --bgp-routing-mode=regional
5
6# Create custom mode VPC network
7gcloud compute networks create my-custom-network \
8 --subnet-mode=custom \
9 --bgp-routing-mode=global
10
11# Create subnet in custom VPC
12gcloud compute networks subnets create my-subnet \
13 --network=my-custom-network \
14 --region=us-central1 \
15 --range=10.0.1.0/24 \
16 --secondary-range=pods=10.1.0.0/16,services=10.2.0.0/20
17
18# Create private subnet (no external IPs)
19gcloud compute networks subnets create private-subnet \
20 --network=my-custom-network \
21 --region=us-central1 \
22 --range=10.0.2.0/24 \
23 --enable-private-ip-google-access \
24 --no-enable-flow-logs
25
26# List networks and subnets
27gcloud compute networks list
28gcloud compute networks subnets list --network=my-custom-network
29
30# Expand subnet (only increase size)
31gcloud compute networks subnets expand-ip-range my-subnet \
32 --region=us-central1 \
33 --prefix-length=23

Intermediate

2gcloud compute routes list --filter="network=my-custom-network"

4# Create custom route to VPN gateway
5gcloud compute routes create vpn-route \
6 --network=my-custom-network \
7 --destination-range=192.168.1.0/24 \
8 --next-hop-vpn-tunnel=vpn-tunnel-1 \
9 --priority=100
10
11# Create route to instance as next hop
12gcloud compute routes create instance-route \
13 --network=my-custom-network \
14 --destination-range=10.10.0.0/16 \
15 --next-hop-instance=my-router-instance \
16 --next-hop-instance-zone=us-central1-a \
17 --priority=200
18
19# Create route to internet gateway
20gcloud compute routes create internet-route \
21 --network=my-custom-network \
22 --destination-range=0.0.0.0/0 \
23 --next-hop-gateway=default-internet-gateway \
24 --priority=1000
25
26# Create peering route
27gcloud compute routes create peering-route \
28 --network=my-custom-network \
29 --destination-range=172.16.0.0/16 \
30 --next-hop-peering=peering-name \
31 --priority=50
32
33# Delete custom route
34gcloud compute routes delete vpn-route --quiet
35
36# View effective routes for an instance
37gcloud compute instances get-serial-port-output my-instance \
38 --zone=us-central1-a \
39 --port=1 | grep "Routes"
🛣️ Route Types & Priorities
  • Subnet Routes: Auto-created for each subnet (priority 1000)
  • Custom Routes: User-defined routes (priorities 0-65535)
  • Peering Routes: Routes from VPC peering connections
  • VPN Routes: Routes through VPN tunnels
  • Internet Routes: Routes to external destinations

Firewall Rules & Security

Intermediate

Firewall rules control inbound and outbound traffic to your VPC network. GCP firewall rules are stateful and can be applied at the network level with hierarchical inheritance.

1# Create allow rule for HTTP/HTTPS
2gcloud compute firewall-rules create allow-web \
3 --network=my-custom-network \
4 --allow=tcp:80,tcp:443 \
5 --source-ranges=0.0.0.0/0 \
6 --target-tags=web-server \
7 --description="Allow HTTP and HTTPS traffic"
8
9# Create internal network rule
10gcloud compute firewall-rules create allow-internal \
11 --network=my-custom-network \
12 --allow=tcp:0-65535,udp:0-65535,icmp \
13 --source-ranges=10.0.0.0/8 \
14 --description="Allow all internal traffic"
15
16# Create SSH rule with specific source
17gcloud compute firewall-rules create allow-ssh \
18 --network=my-custom-network \
19 --allow=tcp:22 \
20 --source-ranges=203.0.113.0/24 \
21 --target-tags=ssh-access \
22 --description="Allow SSH from specific IP range"
23
24# Create deny rule (higher priority)
25gcloud compute firewall-rules create deny-bad-traffic \
26 --network=my-custom-network \
27 --action=deny \
28 --rules=tcp:23 \
29 --source-ranges=0.0.0.0/0 \
30 --priority=10 \
31 --description="Block Telnet traffic"
32
33# Create egress rule
34gcloud compute firewall-rules create allow-egress \
35 --network=my-custom-network \
36 --action=allow \
37 --direction=egress \
38 --rules=tcp:443 \
39 --destination-ranges=0.0.0.0/0 \
40 --target-tags=https-only \
41 --description="Allow HTTPS outbound traffic only"
42
43# List firewall rules
44gcloud compute firewall-rules list --filter="network=my-custom-network"
45
46# Update firewall rule
47gcloud compute firewall-rules update allow-web \
48 --source-ranges=10.0.0.0/8,192.168.0.0/16
🛡️ Firewall Best Practices
  • Least Privilege: Only allow necessary ports and sources
  • Use Tags: Apply rules to specific instances with network tags
  • Hierarchical Rules: Network-level rules apply to all instances
  • Regular Audits: Review and update rules periodically
  • Priority Management: Lower numbers = higher priority (0-65535)
