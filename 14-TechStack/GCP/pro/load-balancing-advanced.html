Load Balancing Overview

Basic

Google Cloud Load Balancing is a fully distributed, software-defined managed service that distributes user traffic across multiple instances of your applications. It enables high availability, auto-scaling, and robust performance for your applications.

⚖️ Key Capabilities
  • Global Load Balancing: Distribute traffic worldwide with single anycast IP
  • Auto-scaling: Automatically scale backend capacity based on load
  • Health Checking: Continuous monitoring of backend health
  • SSL Termination: Built-in SSL certificate management
  • Layer 7 Routing: Content-based routing and URL maps
🔧

Load Balancer Types & Use Cases

Basic

GCP offers different types of load balancers optimized for specific use cases, from global HTTP(S) load balancing to regional network load balancing.

🌐
Global HTTP(S) LB
Layer 7 load balancing for web applications with global distribution
🏢
Regional HTTP(S) LB
Layer 7 load balancing within a single region
⚡
Network TCP/UDP LB
Layer 4 load balancing for TCP/UDP traffic
🔄
Internal TCP/UDP LB
Layer 4 load balancing for internal traffic
🔒
SSL Proxy LB
Layer 4 SSL termination for encrypted TCP traffic
🚀
TCP Proxy LB
Layer 4 proxy for TCP traffic without SSL termination
Creating a Global HTTP Load Balancer
bash
1# 1. Create instance template
2gcloud compute instance-templates create lb-backend-template \
3 --region=us-central1 \
4 --network=default \
5 --subnet=default \
6 --tags=http-server \
7 --image-family=debian-11 \
8 --image-project=debian-cloud \
9 --metadata=startup-script='#!/bin/bash
10apt-get update
11apt-get install -y apache2
12echo "Hello from $(hostname)" > /var/www/html/index.html'
13
14# 2. Create managed instance group
15gcloud compute instance-groups managed create lb-backend-group \
16 --template=lb-backend-template \
17 --size=2 \
18 --zone=us-central1-a
19
20# 3. Create firewall rule
21gcloud compute firewall-rules create fw-allow-health-check \
22 --network=default \
23 --action=allow \
24 --direction=ingress \
25 --source-ranges=130.211.0.0/22,35.191.0.0/16 \
26 --target-tags=http-server \
27 --rules=tcp:80
28
29# 4. Create health check
30gcloud compute health-checks create http http-basic-check \
31 --port=80
32
33# 5. Create backend service
34gcloud compute backend-services create web-backend-service \
35 --protocol=HTTP \
36 --port-name=http \
37 --health-checks=http-basic-check \
38 --global
39
40# 6. Add backend to service
41gcloud compute backend-services add-backend web-backend-service \
42 --instance-group=lb-backend-group \
43 --instance-group-zone=us-central1-a \
44 --global
45
46# 7. Create URL map
47gcloud compute url-maps create web-map \
48 --default-service=web-backend-service
49
50# 8. Create target HTTP proxy
51gcloud compute target-http-proxies create http-lb-proxy \
52 --url-map=web-map
53
54# 9. Create forwarding rule
55gcloud compute forwarding-rules create http-content-rule \
56 --address=lb-ipv4-1 \
57 --global \
58 --target-http-proxy=http-lb-proxy \
59 --ports=80

Global Load Balancing with Cross-Region Failover
🌍 Global vs Regional Comparison

🌐 Global Load Balancing

  • Single anycast IP address
  • Automatic failover across regions
  • Content-based routing (Layer 7)
  • Global CDN integration
  • Higher cost but better performance

🏢 Regional Load Balancing

  • Regional IP address
  • Failover within region only
  • Network-level routing (Layer 4)
  • Lower latency within region
  • Cost-effective for regional apps

Health Checks & Backend Monitoring

Intermediate

Health checks continuously monitor the status of your backend instances, ensuring that traffic is only sent to healthy instances and automatically routing around failed instances.

🌐
HTTP/HTTPS
Web application health checks
🔌
TCP
Network-level connectivity checks
📊
UDP
UDP service availability checks
💓
SSL
SSL certificate validation
🔍
Custom
Application-specific health logic
⚡
Legacy
Backward compatibility checks
Configuring Advanced Health Checks
bash
1# Create HTTP health check with custom parameters
2gcloud compute health-checks create http advanced-health-check \
3 --port=8080 \
4 --request-path=/health \
5 --check-interval=10 \
6 --timeout=5 \
7 --healthy-threshold=2 \
8 --unhealthy-threshold=3 \
9 --host=example.com
10
11# Create HTTPS health check
12gcloud compute health-checks create https https-health-check \
13 --port=443 \
14 --request-path=/api/health \
15 --proxy-header=NONE
16
17# Create TCP health check
18gcloud compute health-checks create tcp tcp-health-check \
19 --port=6379 \
20 --proxy-header=NONE
21
22# Create custom health check with response validation
23gcloud compute health-checks create http custom-health-check \
24 --port=80 \
25 --request-path=/health \
26 --response=OK \
27 --host=my-app.example.com
