# BigQuery ML Roadmap – 6 Weeks

## Weeks 1-2: Foundations (🟢)
- [ ] Day 1: CREATE MODEL basics (linear/logistic)
- [ ] Day 2: ML.EVALUATE; train/val/test splits
- [ ] Day 3: ML.PREDICT; batch predictions
- [ ] Day 4: Data prep: partitions/clusters; avoid SELECT *
- [ ] Day 5: Feature selection; leakage checks
- [ ] Day 6: Time series intro (ARIMA+)
- [ ] Day 7: Mini-project: basic classifier
- [ ] Day 8: Metrics interpretation (precision/recall/ROC)
- [ ] Day 9: Costs: slot usage; query bytes
- [ ] Day 10: Review + refactor
- [ ] Day 11-12: Explainability: EXPLAIN_PREDICT, GLOBAL_EXPLAIN
- [ ] Day 13-14: Model versioning via naming/datasets

**Milestone**: Train/evaluate/predict with clean data and basic metrics.

## Weeks 3-4: Intermediate (🟡)
- [ ] Day 15: Boosted trees; hyperparameter tuning (ML.TRIALS)
- [ ] Day 16: Matrix factorization/recommendations
- [ ] Day 17: K-means clustering; eval
- [ ] Day 18: Time series tuning; holiday effects
- [ ] Day 19: Scheduled queries for retrain; automation
- [ ] Day 20: Data quality checks pre-train
- [ ] Day 21: Export models; remote models; Vertex integration
- [ ] Day 22-23: Mini-project: tuned model + scheduled retrain
- [ ] Day 24-25: Governance: IAM on datasets/models; authorized views
- [ ] Day 26-28: Cost tuning: sample for dev, prune partitions

**Milestone**: Automated, tuned models with governance basics.

## Weeks 5-6: Advanced (🔴)
- [ ] Day 29: Drift monitoring; metric comparison per version
- [ ] Day 30: Canary predictions; rollback strategy
- [ ] Day 31: Feature Store integration for consistency
- [ ] Day 32: Security: CMEK, row/col security
- [ ] Day 33: Compliance: audit logs, lineage docs
- [ ] Day 34: Performance/load testing on predictions
- [ ] Day 35: Multi-model management; naming conventions
- [ ] Day 36: CI/CD for SQL models (lint/tests)
- [ ] Day 37-38: Capstone: governed, monitored BQML pipeline
- [ ] Day 39-42: Documentation, SOPs, cost/perf reports

**Milestone**: Governed, monitored BQML with automation and rollbacks.
