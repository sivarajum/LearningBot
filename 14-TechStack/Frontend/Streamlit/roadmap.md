# Streamlit Roadmap – 6 Weeks

## Weeks 1-2: Foundations (🟢)
- [ ] Day 1: Install; run `streamlit hello`
- [ ] Day 2: Widgets (text_input, selectbox, button)
- [ ] Day 3: Layout (columns, sidebar, tabs)
- [ ] Day 4: Session state basics
- [ ] Day 5: Simple charts (line/bar) built-ins
- [ ] Day 6: File upload; caching intro
- [ ] Day 7: Forms vs free widgets
- [ ] Day 8: Theming via config.toml
- [ ] Day 9: Mini-project: input → compute → chart
- [ ] Day 10: Deploy locally behind reverse proxy
- [ ] Day 11-12: Secrets via st.secrets; env handling
- [ ] Day 13-14: Review + refactor

**Milestone**: Build a small app with forms, state, charts.

## Weeks 3-4: Intermediate (🟡)
- [ ] Day 15: cache_data/resource for I/O-heavy work
- [ ] Day 16: Data prep with pandas; altair/plotly charts
- [ ] Day 17: Multi-page apps (pages/ directory)
- [ ] Day 18: File handling; validation; limits
- [ ] Day 19: API calls with retries/timeouts
- [ ] Day 20: Authentication pattern (proxy or custom)
- [ ] Day 21: Error handling; user-friendly messages
- [ ] Day 22-23: Mini-project: multi-page dashboard with caching
- [ ] Day 24-25: Performance checks; avoid rerun storms
- [ ] Day 26-28: Packaging app in Docker; config via env

**Milestone**: Production-ready multi-page app with caching and auth.

## Weeks 5-6: Advanced (🔴)
- [ ] Day 29: Custom components; integrate external JS
- [ ] Day 30: Observability: logging, timing metrics
- [ ] Day 31: Security: sanitize inputs, limit uploads, guard timeouts
- [ ] Day 32: Feature flags/config per env
- [ ] Day 33: CI/CD: tests, lint, container build
- [ ] Day 34: Cost/perf: cache TTLs, memoization
- [ ] Day 35: Disaster readiness: backups for stateful deps
- [ ] Day 36: DR/rollback plan for deployments
- [ ] Day 37-38: Capstone: full app with auth, cache, charts, CI/CD
- [ ] Day 39-42: Documentation and runbooks

**Milestone**: Observable, secure, deployable Streamlit app with CI/CD.
